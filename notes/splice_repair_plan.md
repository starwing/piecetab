# Linecache Splice 区间删除终极重构计划

## 1. 危机与理论来龙去脉

在实现并测试经典的度量 B+ 树（Metric B+ Tree）进行大规模 Range Delete (`splice` / `clearbreaks`) 时，我们遭遇了深层悬垂下无法治愈的 Underfull 结构。原有的算法及通常粗糙的修改都惨遭失败。

### 为什么一次或两次通过（传统 Bottom-Up 或 直白合并）必然失败？

**反例展现：深层两端保留 `[1..1000] 删除内含至 [1, 1000]`**
- 假设这是一棵4层的树，我们分歧层（Divergence）在第0层（根），并需要保留极左的叶子1和极右的叶子1000。中间的子树全部被直接挖空。
- **纯 Bottom-Up 合并困境**：当你在 Leaf 层（第3层）将 `叶子1` 和 `叶子1000` 两端“相邻”的心智模型合并时，它们合成了一个只有 2 个子项（严重低于 FANOUT/2）的不足叶子。但在传统的 B+ 树自底向上重新平衡算法中，补齐必须要找同父兄弟。由于它们的父亲（第2层的残存节点）因为中间子树全被删除，此刻也只剩下这唯一的单传残脉。既无同父兄弟，如何同侧借位？
- **传统解法死锁**：若放任其 Underfull 留存给上层合并解决，上层（第2层）合并甚至去向上一层借位后，第2层获取了健康完整的子节点，但原本留在第3层的那个只有 2 个项的严重不足子树**已被巡查探过，不再受上层福泽反哺**！这种依靠单次 Bottom-up 的方法注定留下“永远填不平的底坑”。
  
**“底求顶放”的本质与前瞻倒流的破局：**
- 正如研究表明的那样，真正能够解决跨层借位的算法流往往是一个隐形的嵌套：即底层发现无兄弟可依，于是抛出合并需求，递归上层（向父亲要）；若父亲亦不足，父亲向上求爷爷；待上峰求得横向亲戚并把新资源揽入麾下后，再把借来的健康资源灌输分发给下层。这就是**“底层探测求要 → 顶层兼并 → 自上顺流补血”**的本质。
- 然而，我们既已明晰在区间删除中，**从分歧层 `l` 斜切向下的那条边缘线（即左边缘及右边缘）必然全部遭到重创（全部都处于空壳或偏瘦状态）**，那么我们何苦还要任由它们层层往上发起栈式“打报告”呢？我们完全可以采取前瞻性降维打击（Proactive Top-Down）：即从必定富足（或者有途径富足）的分歧父本 `l` 开始，自带着资源往下层“开仓放粮”，这样所有空洞都能在一遍顺下的 `for` 循环中即时获得新兄弟并填满。

## 2. 终极三段法重构框架

我们将原本无序修补的 `lcD_splicerange` 完全收敛到最为清晰的等效理论模型：“底切顶收、上顺分粮、下归落定”。

### 第一段：断支残脉（Prune & Trim）
**目标：截断无用组织，确立空洞左右崖壁与结构骨架**
1. **Divergence (`l`)**：探测游标 C1 和 C2 首次分道的层级。
2. **Prune (`l`层剪中段)**：对中间被跨越包含的无辜巨根尽数摧除，这使得 C1 和 C2 变成悬崖边缘，此时它们已成为 `l` 下相邻的同门脉系。
3. **TrimLeaf**：严格执行末端叶子段的数据消费计算并清空余量（必须要有完善的左剪右并移裁测试）。
4. **TrimNode（关键免转移优化）**：无需发生任何 `memmove` 去挤压对齐！只需简单利用 C1 和 C2 的 paths 和 idx 得知哪些分支属于留存骨架。让移除退化成指针界限的偏移标记，将彻底归拢合并推迟到顺流之时。

### 第二段：顶流输血（Proactive Top-Down Repair）
**目标：以简单循环规避深递归栈，将资源向下沿崖壁降级补给（只借补，不开花扩高）**
从 `d = l+1` 顺延至 `levels-1` 递降：
- 此时 `C1` 及 `C2` 边缘节点在 `d-1` 层已经与健康兄弟并立（由上级事前调停完毕，绝无单传空巢）。
- `C1` 节点调用专门的辅助方法（如 `lcD_ensure_child_fill`），由于其必定能在 `d-1` 内部找到强壮兄弟（向本家的右侧健康方借树，甚至直接吞并），从而使 `C1` 及其底下的路径骤得充足资源。
- `C2` 节点同理（向左侧堂亲借）。
- 这种修补从上逐层进行，当修到 $d+1$ 时，$d+1$ 会惊异地发现自己原本空空如也，却由于上面的调转而在左右得到了无数饱满堂族。如此推及叶底层，一切边缘枯叶皆受荫借，瞬间肥瘦均衡、合归 B+ 算法正统。

### 第三段：归本扶正（Bottom-Up Rebalance & Root Shrink）
**目标：修复宗主（分歧点及以上）的可能欠缺与调整树总高**
- 二段下放必然从宗支（`l` 层左右及 `d-1` 层借出端）抽散了节点数。因此最后只需以标准的、单点专用的自底向上 `Rebalance`，从 $l$ 层向根部检查（向同父堂亲借合并）。
- 如果根节点经过抽身或缩并变得唯余其一，连斩降根直至健康多子，这亦是树高因大区间拔除而合理降阶的必由途径。

## 3. 具体函数行为与示例规范

### 1. `lcD_trimleaf(lc_Cursor* C, int left, size_t *pdel)`
- **明确边界：** 叶级切割不涉及重新平衡，它是数据移除和统计的开端。
- **参数示例与左修法则：** `left=1` 代表这是 `C1`（切割后游标及之后数据抛却，保留其左面）。则只需销毁 `lidx` 右方各段。
- **右修移归与消费补充：** `left=0` 则是 `C2`（保留其右面）。此极必须发生物理移动，将留存的数据向前填满；其抹除的左部 `bytes` 必须并入 `*pdel` 提供外层汇报消耗。

### 2. `lcD_ensure_child_fill(lc_Node* parent, unsigned child_idx)` (新增重器)
- **行为：** B树中的经典 `borrow/merge` 方法。它观测 `parent->children[child_idx]` 的度数，如果低于 `FANOUT/2`。首先向 `child_idx-1` 或 `child_idx+1` 发起乞求借入（并维护 metrics）；若均紧巴巴，则将它们二者合二为一（导致 `parent` 减子）。
- **特调游标更替：** 当针对 `C1` 或 `C2` 的节点执行充填缩消且发生了合并，游标的 `paths` 地址或 `idx` 会改变！该函数必须具备追踪修复游标之能。

### 3. `lcD_splicerange(lc_Cursor* C1, lc_Cursor* C2)`
- 这是三段法的灵魂调度池，它的执行流只用最本分的三大循环组成：
```c
// 第一段：断支定界
lcD_prune(C1, C2, divergence_l);
// (仅对边界做标识和处理叶级消耗)

// 第二段：自上向下前瞻修复 (Proactive Top-Down)
for (int d = divergence_l + 1; d < levels; d++) {
    lcD_ensure_child_fill(parent_at_d_minus_1, idx_c1);
    lcD_ensure_child_fill(parent_at_d_minus_1, idx_c2);
    // 更新 C1, C2 的游标...
}

// 并在此底层进行可能存在的多余空壳残余的底消...

// 第三段：常规底部追封与缩根
lcD_rebalance(divergence_l);
while (root_has_only_one_child) { shrink_root(); }
```

## 4. 实施细分步骤

- **Step 1: 先斩叶（TDD of `trimleaf`）**。优先对现有代码中的 `trimleaf` 补全单测并修复物理问题（必须独立成军，通过诸如切割非整片区域、空叶回收边界等测试），确保对上游交代正确的消耗偏移差。
- **Step 2: 实作顶层下输池（Core Merge/Borrow）**。编写无刺状态的 `lcD_ensure_child_fill`，并在这一步把 C2 的 `trimnode` 转变为“免搬优化（只拨指针限界）”。
- **Step 3: 安装调度及收尾纠根（Assemble 3-pass & Test Coverage）**。编织 `splicemerge` 主环。并在 `linecache_test.c` 里加回那曾令旧思路坠渊的 `[1..1000] -> [1, 1000]` 深谷大峡谷悬切反例，验证其借兄弟之力如流水顺抵，平稳过渡；再测试多层多列跨越合并的情形以查无漏网孤儿。调试全过。

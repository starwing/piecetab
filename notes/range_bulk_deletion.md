## 1. 算法概述与核心哲学

在传统的 B+ 树范围内执行大批量删除时，如果逐个删除会导致 $O(K \log N)$ 的极高代价，且会引发频繁的级联重平衡。本算法针对**计量B+树**（依靠按节点/行统计的 bytes/breaks 进行路由定位，无需严格维护传统路由 Key）的特性，设计了一种**时间复杂度绝对稳定在 $O(\log N)$** 的区间批量删除算法。

**核心架构哲学：彻底解耦“拓扑减法”与“容量平移”**

*   **Bottom-Up（自底向上）只做纯减法（Trim & True Merge）：** 在修剪区间废弃节点时，仅在容量允许时执行**彻底合并**。绝对禁止在这一阶段发生兄弟节点间的“均摊/平分（Redistribute）”。这完美锁死了左边界 $L$ 的物理路径（Cursor 绝对不偏移），避免了跨子树操作导致的路径断裂悖论。
*   **Top-Down（自顶向下）只做纯平移（Redistribute & Rebalance）：** 利用自底向上留下的“未合并节点容量和必定 $> FANOUT$”的数学铁律，在确定了公共祖先之后，自顶向下通过**平分**恢复节点合法性。通过“上层富余容量”阻断 B+ 树的合并级联。

## 2. 术语与变量定义

*   `levels`: 树的叶子层索引（Root 为 $0$ 层）。
*   `L` / `R`: 待删除区间的左边界（Left）和右边界（Right）的游标。
*   `C->paths`: 游标`C`自顶向下的寻址路径数组。`paths[dl]` 代表游标在 $dl$ 层所在的物理节点（指向 $C_{dl-1}$ 的 children 中某个项的指针）。$C_{dl}$ 表示 `*C->paths[dl]`
*   `l`: `L` 和 `R` 的最近公共祖先层（Lowest Common Ancestor，LCA）。
*   `FANOUT`: 节点的最大容量。合法节点容量要求 `>= FANOUT/2`。

## 3. 函数原语定义（严格区分合并与平分）

为了保证算法不产生变型，原有的 `merge` 逻辑必须被严格拆分为两组职责单一的函数：

1.  **Bottom-Up 专用（只合并，不平分）：**
    *   `absorbleaf(L, R)` / `absorbnode(L, R, dl)`：
        *   **逻辑**：当且仅当 `|L| + |R| <= FANOUT` 时，将 `R` 中所有数据移动到 `L` 末尾，累加计量 metrics，右移 $R_{dl}$，使`trimnode(R, 1)` 能后续删除这个节点。
        *   **如果 `> FANOUT`**：直接返回，不做任何操作，允许 `L` 和 `R` 处于残缺状态。
2.  **Top-Down 专用（只平分，绝对不销毁）：**
    *   `balancenode(L, R)` / `balancenode(L, R， dl)`：
        *   **逻辑**：在 `L` 和 `R` 之间平均分配数据，使得两者均 `>= FANOUT/2`，并更新计量 metrics。需要断言 `|L| + |R| > FANOUT`。

## 4. 算法执行步骤详解

### 第一阶段：寻址与快速路径
1. 查找边界 `L` 和 `R` 的路径，分别记为 `L->paths` 和 `R->paths`。
2. 自顶向下比对路径，找到两者的共节点层 `l`。
3. 如果 `l == levels`（在同一叶子内），调用单节点叶内删除算法，返回。

### 第二阶段：Bottom-Up 纯减法修剪（自叶子至 `l+1` 层）
*   **操作叶子层（`levels`）：**
    *   `trimleaf(L, right)`：清理 `L` 属于删除区间的内容。
    *   `trimleaf(R, left)`：清理 `R` 属于删除区间的内容。
    *   `absorbleaf(L, R)`：尝试真合并。如果合并，`R` 被销毁。
*   **操作内部节点（游标 `dl` 从 `levels` 到 `l+1`）：**
    *   `trimnode(L, dl, right)`：释放游标右侧所有子树。
    *   `trimnode(R, dl, left)`：释放游标左侧所有子树。（注意：如果 `dl+1` 层返回 `R` 已被销毁，需将 `R` 当前指向的废弃指针一同修剪移除）。
    *   `absorbnode(L, R, dl-1)`：尝试真合并。如果不满足条件则保留现状。

> **架构防御点：** 此时 `L` 和 `R` 可能属于完全不同的深层子树。因为我们**绝对禁止了平分（Redistribute）**，所以数据绝对不会从 `L` 流向 `R`。`L` 的物理地址岿然不动，`L->paths` 的所有父级指针依然完美合法，彻底消除了 Cursor Jump（游标跳跃）和拷贝路径导致的崩溃问题。

### 第三阶段：LCA 修剪与顶层“蓄水”（`l` 层）
*   **断桥（Prune）：** 对共节点层 `l` 执行 `prune(L, R, l)`，彻底删除介于 `L` 和 `R` 指针之间区间内的所有子树。此时 `L` 的子树和 `R` 的子树正式成为相邻的“亲兄弟”。
*   **特殊缩根：** 如果 `l == 0` 且修剪后 Root 只有一个孩子，执行缩根操作。
*   **顶层蓄水（强制 Fat Node）：** 
    此时 $L_l$ 可能残缺。但由于 `L` 处于区间最左边缘，其左侧兄弟（`L-1`）绝对是合法且未被触碰的。
    强行让 $L_l$ 与左兄弟（或 $R_l$）执行标准的 B+ 树合并/平分操作，**确保操作后 $L_l$ 的孩子数严格 `> FANOUT/2`**。
    *(如果此处导致 `l-1` 层少一个孩子，则触发标准的向上传递 rebalance)*。

### 第四阶段：Top-Down 平移修复与游标重塑（自 `l+1` 层至 `levels`）
*   游标 `dl` 从 `l+1` 层向下迭代到叶子层 `levels`。
*   **情况 A：Bottom-Up 时没有合并（`R` 节点依然存活）**
    *   由于第二阶段没有合并，数学上绝对保证 $|L_{dl}| + |R_{dl}| > FANOUT$。
    *   执行 `balancenode(L, R)`（底层为 `balancenode`）。
    *   **结果**：数据被平分，两者必定都 $\ge FANOUT/2$（合法）。因为没有节点被销毁，父节点（$dl-1$ 层）不会丢失孩子。
*   **情况 B：Bottom-Up 时发生了真合并（`R` 节点已死，只剩 `L`）**
    *   此时 `L` 包含了所有缝合的数据，但可能 $< FANOUT/2$。
    *   执行标准单点修复 `rebalance(L, dl)`。
    *   **结果**：如果向左兄弟借，父节点不变；如果与左兄弟合并，父节点（`dl-1` 层）会失去 1 个孩子。
    *   **终极阻断**：由于我们在第三阶段（蓄水）和上一轮迭代保证了父节点**严格** `> FANOUT/2`，即便现在扣减 1 个，父节点依然 `>= FANOUT/2`！**级联合并被完美阻断，绝对不会向上回溯。**
*   **游标同步**：发生数据平分/借用后，实时更新当前层 `L->paths[dl]` 的游标偏移（基于最新的 metrics 累加值定位）。由于是顺藤摸瓜自顶向下，即使数据发生物理转移，游标也能精准同步，直到叶子层。

## 5. 算法正确性与优势证明

1. **规避“两棵单链树”陷阱（单链反例证明）：**
   假设删除导致左右树变成深度极高但每层仅剩 1 个节点的单链。如果在 Bottom-Up 不做任何操作，Top-Down 时会误以为拥有两个有效分支，导致容量计算崩溃。
   本方案中，因为 $|L_{levels}| + |R_{levels}| \le FANOUT$，Bottom-Up 阶段会果断执行**真合并**，底层将彻底吃掉 `R`。单链被直接折叠，Top-Down 时只会看到一棵残缺的左树，然后顺理成章地向绝对合法的左兄弟（区间外节点）借用节点，逻辑完美闭环。
2. **零路径悖论（Path Paradox Zeroed）：**
   因为 Bottom-Up 阶段只有“`R` 融入 `L` 并销毁”或“按兵不动”，数据焦点永远不会脱离 `L` 节点，`L->paths` 在物理内存层面的指针依然 100% 正确。不依赖任何危险的 `memcpy(paths)`。
3. **计量（Metrics）一致性：**
   作为计量 B+ 树，Bottom-Up 合并时只需将 `bytes/breaks` 粗暴相加，即可保证路由寻址的总量一致性。Top-Down 下沉修复时，再通过这些正确的上层度量信息，安全地重新切分底层节点。
